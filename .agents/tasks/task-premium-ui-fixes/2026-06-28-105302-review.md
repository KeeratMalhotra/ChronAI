# Premium UI polish: Spotify widget, chat/voice redesign, toasts, TopBar actions, profile upload, cached integration status

Seven UI features landing as a single batch: the Spotify mini-player keeps its iframe always-mounted so music never stops on snap, the gradient AI chat FAB is replaced by a matte TopBar icon, voice mode renders inline inside the chat panel, toasts get a Web Audio pop and spring entrance, quick actions move to a TopBar dropdown, profile pictures can be uploaded via base64/localStorage, and integration status is cached to prevent a flash.

**Watch for:** The Spotify play/pause toggle and volume slider are purely cosmetic — neither controls the embedded iframe (confirmed). Profile picture upload has no file-size guard, letting multi-MB base64 strings saturate localStorage (confirmed). The TopBar "Start Pomodoro" / "Start Focus" actions dispatch a custom event that nothing listens for (confirmed). A new `AudioContext` is created and closed on every toast — on rapid-fire notifications this may hit browser limits (likely).

## High-level view

The Spotify widget solves the unmount problem by rendering the iframe offscreen when snapped (`left: -9999px`) and in-place when expanded. But the click-to-play/pause semi-circle and the expanded volume slider are UI decoration only — there's no programmatic bridge to the Spotify embed's playback state, so the toggle icon will desync from actual playback.

Chat architecture is refactored into the layout: `AIChatPanel` is now mounted once in `dashboard/layout.tsx` rather than per-page, gains a detachable floating-window mode, and the voice overlay is collapsed into an inline swap inside `ChatExperience`. The old full-screen entity canvas is removed (bundle win). The dangling issue is the detached panel's fixed initial position (`calc(50% - 275px)`) which will clip on small viewports.

The toast notification system mints a fresh `AudioContext` per toast and closes it after the 200ms sound. This will fail or produce `NotAllowedError` if toasts arrive in bursts before user gesture unlocks audio.

The profile picture upload reads the file as a base64 data URL and persists it directly to `localStorage` with no size validation. A 10 MB photo produces a ~13 MB base64 string that exceeds most browsers' per-origin localStorage quota (5-10 MB), silently failing or throwing `QuotaExceededError`.

"Start Pomodoro" and "Start Focus" in the TopBar dropdown dispatch `chronai-start-focus` but no component subscribes to that event — clicking them is a no-op.

Integration-status caching reads from `localStorage` immediately on mount then fetches fresh data in the background. This eliminates the disconnected-to-connected visual flash.

## Spotify widget: always-mounted iframe and the control illusion

When snapped, a hidden container (`left: -9999px, opacity: 0, pointerEvents: none`) keeps the iframe in the DOM so the Spotify web player continues streaming. When expanded, a second iframe with the same `src` renders inside the visible card. Two iframes share the same embed URL — the transition from snapped to expanded conditionally renders the second iframe, which may restart playback since Spotify embeds initialize on mount.

The semi-circle button toggles `isPlaying` local state and swaps a Play/Pause icon. There is no `postMessage`, no ref to the iframe's `contentWindow`, no Spotify Web Playback SDK integration. The user sees "Pause" after clicking, but the music continues regardless. This is misleading UX.

The volume slider stores a numeric value in state and renders a styled range input. That value is never sent anywhere — the Spotify embed exposes no volume control API from the host page. The slider is decorative.

## Toast notification sound: AudioContext lifecycle

```typescript
const audioCtx = new (window.AudioContext || ...webkitAudioContext)();
oscillator.connect(gainNode);
gainNode.connect(audioCtx.destination);
oscillator.start(audioCtx.currentTime);
oscillator.stop(audioCtx.currentTime + 0.2);
oscillator.onended = () => { audioCtx.close(); };
```

Chrome limits active `AudioContext` instances; creating more than 6 without closing prior ones logs a warning and may silently fail. The `close()` in `onended` mitigates this, but if toasts fire faster than the 200ms sound duration (e.g., multiple AI suggestions arriving in a batch), contexts stack up before prior ones close. A single module-level context with a shared gain node would be simpler and safer.

Browsers also require a user gesture before `AudioContext` can produce sound. If the first toast fires before the user has clicked anything on the page, the context will be created in a suspended state and the sound will be silent. The `try/catch` swallows this, so it degrades gracefully but produces no audio.

## Profile picture: localStorage as an image store

```typescript
const handleProfilePictureSelect = (e) => {
  const file = e.target.files?.[0];
  if (!file) return;
  const reader = new FileReader();
  reader.onload = () => {
    const base64 = reader.result as string;
    setCustomProfilePicture(base64);
    localStorage.setItem("chronai-profile-picture", base64);
  };
  reader.readAsDataURL(file);
};
```

No validation on `file.size`, `file.type` (beyond the advisory `accept="image/*"` attribute), or image dimensions. A 5 MB DSLR photo produces a ~6.7 MB base64 string. Most browsers cap localStorage at 5 MB per origin. Exceeding this throws a `QuotaExceededError` that's unhandled — `localStorage.setItem` throws, but `setCustomProfilePicture` has already updated state (so the image shows for the current session but disappears on reload). This is a silent data-loss bug for users with large photos.

## TopBar quick actions: dead event dispatch

The "Start Pomodoro" and "Start Focus" buttons dispatch `window.dispatchEvent(new CustomEvent("chronai-start-focus"))`. In `dashboard/page.tsx`, `FocusMode` is controlled by `focusActive` state and `setFocusActive` — but there's no `useEffect` listening for the custom event. The old `CommandPalette` component (removed in this diff) called `onFocusMode` directly; the new TopBar path has no equivalent wiring. Clicking either action closes the dropdown and does nothing visible.
